# Nobody — AI Security Partner

> no0boy 的安全搭档。你学什么，它学什么。你变强，它变强。

---

## 定位

**不是聊天机器人，不是 Demo。**

是一个会思考、会调用技能、会记住每个人、会持续成长的个人安全智能体。

---

## 权限模型

```
Nobody 三层隔离

┌───────────────────────────────┐
│  公共知识层 (knowledge/public/) │  nobody(主人)可写，所有人只读
├───────────────────────────────┤
│  私人记忆 (memory.db)          │  按用户隔离，每人独立空间
├───────────────────────────────┤
│  游客模式                      │  只能搜索公共知识，不留任何记忆
└───────────────────────────────┘
```

## 版本迭代

### v1.5 — 真向量 RAG

**思路**：v1.0 的关键词检索不懂语义，"被黑"搜不到"入侵"。

**实现**：
- 文本切片(500字) → ChromaDB 内置 Embedding 生成向量 → 入库
- 检索时向量相似度 + BM25 关键词混合，RRF 融合排序
- 首次启动自动下载 Embedding 模型

### v1.4 — 公共知识层 + 游客模式

**思路**：面试官或新用户不需要注册，点了就能用。但不能污染任何人的记忆。

**实现**：
- `knowledge/public/` 存放 no0boy 的经验库，启动时自动索引
- 游客端点 `/api/auth/guest`：免注册，无记忆写入
- 所有人可搜索公共知识，只有 nobody 能修改

### v1.3 — 五维记忆 + 用户注册

**思路**：Nobody 不能一问三不知——要记住每个人是谁、什么方向、学到哪了。

**实现**：
- SQLite `memory.db` 持久化存储
- 五维记忆：👤身份 ❤️偏好 📚学习 💼经验 🎯目标
- 按 `user_id` 隔离，每人只能看到自己的记忆
- 用户注册/登录系统

### v1.2 — 技能激活

**思路**：技能不能只是 JSON 标签，要真正干活。

**实现**：
- `match_skills()` 匹配到的技能执行对应的工具函数
- `sqli_detect`、`cve_lookup` 等函数注册表
- 前端聊天页显示当前激活的技能标签

### v1.1 — 安全 Router

**思路**：不是所有问题都一样紧急。"服务器被入侵"应该比"SQL注入是什么"更优先。

**实现**：
- `assess_severity()` 函数，关键词快速判断 P0/P1/P2/INFO
- 前端对话页红色/橙色/黄色/绿色严重度标签
- 不同严重度对应不同响应策略

### v1.0 — 安全问答 Agent

**思路**：基于校园 AI 平台已验证的 Agent 框架，迁移到安全领域。

**实现**：
- 复用 `hybrid_search.py`、`token_tracker.py`
- 安全知识库：OWASP Top 10 / SQL 注入指南 / 应急响应
- 2 个 Agent：威胁研判 + 漏洞研究
- 2 个技能：CVE 查询 + SQL 注入分析
- 配置驱动：`persona.json` 定义人格，`provider.json` 切换模型

## 目录结构（配置驱动）

```
ai助手/
├── brain.py              ← 核心引擎
├── persona.json          ← 人格定义
├── provider.json         ← 模型配置 (${ENV_VAR} 安全注入)
│
├── agents/               ← 一个 json = 一个 Agent
├── skills/               ← 一个 json = 一个技能（插件化）
├── knowledge/
│   ├── seeds/            ← 种子文档
│   └── public/           ← 公共知识层（nobody 维护，所有人只读）
├── memory.db             ← 五维记忆（SQLite，按用户隔离）
│
├── api/                  ← 路由层
│   ├── auth.py           ← 认证（注册/登录/游客）
│   ├── chat.py           ← 聊天接口
│   └── webhook.py        ← 外部渠道（飞书等）
│
├── services/             ← 核心服务（从校园 AI 平台复用）
│   ├── hybrid_search.py  ← 混合检索（向量+BM25+RRF）
│   └── token_tracker.py  ← 成本监控
│
├── frontend/             ← 前端
│   ├── login.html        ← 登录页（三入口）
│   └── index.html        ← 聊天页
│
└── README.md
```

## 核心原则

- 配置驱动，不写死
- 技能插件化，加能力不改核心
- 大脑与模型解耦（provider.json 一行切换）
- 公共知识所有人只读，私人记忆按用户隔离
- 小步迭代，每版本只加一个核心能力
